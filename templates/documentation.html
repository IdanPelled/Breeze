    Breeze Programming Documentation

    Breeze is a fun, kid-friendly programming language. This documentation serves as a guide to help you get started with Breeze, learn its syntax and core concepts, and unleash your creativity by solving problems through coding.



    Data Types

    In programming, data types are like different categories or kinds of information that we can work with. They help us define and understand the type of data we are dealing with. Here are a few common data types:

    • Numbers: Numbers are used to represent numerical values. They are whole numbers (like 5, -10, 100). In programming, we use numbers for calculations, counting, and measuring things.
    • Text or texts: Text or texts are used to represent sequences of characters, such as letters, words, or sentences. For example, your name, "Hello, World!", "I love coding" are all examples of texts. We use text or texts to store and work with words, sentences, and any kind of textual information.
    • Booleans: Booleans are a special data type that represent either true or false. It's like answering a yes-or-no question. For example, "Is it raining outside?" can be answered with either true (yes, it's raining) or false (no, it's not raining). Booleans are helpful when we want to make decisions in our programs or check if something is true or false.

    Why do we need data types? Well, imagine if you asked someone for their age, and they replied with "Elephant." That wouldn't make much sense, right? Data types help us make sure we're using and storing the right kind of information. They also determine how we can interact with that information. For example, we can add two numbers together or concatenate (join) two texts, but we can't add a number and a text together.

    When we write programs, we use data types to define the kind of information we want to work with. We declare variables, which are like labeled containers that can hold different data types. For example, we might have a variable called "score" that stores a number, or a variable called "name" that stores a text.

    Understanding data types helps us write programs that make sense and work correctly. It's like organizing our thoughts and information in a way that the computer can understand and process.

    Literals

    In Breeze, literals are fixed values that are directly written within the code and represent specific types of data. Here are the literals available in Breeze for the different data types:

    Boolean Literals:

    • no: Represents the boolean value of "false."
    • yes: Represents the boolean value of "true."

    Boolean literals are used to express logical conditions or true/false values in Breeze.

    Number Literals:

    Number literals in Breeze represent numeric values. They can be positive or negative integers without decimal places. Examples of number literals include 0, 123, -5, etc.

    Number literals are used in mathematical calculations, comparisons, or whenever a numeric value is required.

    Text Literals:

    Text literals in Breeze represent strings of characters or words. They are enclosed within double quotation marks ("). Examples of text literals include "Hello", "Breeze", "123", etc.

    Text literals are used for displaying messages, storing textual data, or providing prompts for user input.

    These literals provide a way to express specific values directly in the code without the need for variables or calculations. They are useful for assigning fixed values or specifying conditions in your Breeze programs.

    It's important to note that literals are different from variables. While literals represent fixed values, variables are used to store and manipulate data that can change during the execution of a program.

    Variables

    In programming, variables are like labeled containers that can hold different types of information. They allow us to store and manipulate data in our programs. Just like in the real world, where you might have a box labeled "toys" to hold your toys or a jar labeled "cookies" to hold your cookies, variables in programming give us a way to store and refer to different pieces of information.

    Here's how variables work:

    • Naming Variables: To use a variable, we give it a name that represents the information it will hold. For example, we might have a variable called "score" to keep track of a player's score in a game.
    • Assigning Values: We assign a value to a variable by using the "set" keyword followed by the variable name and the desired value. For example, we can assign the value 10 to the variable "score" to represent a starting score of 10.
    • Updating Values: As our program progresses, we can update the value of a variable by assigning it a new value. For example, if the player earns more points, we can assign a higher number to the "score" variable to reflect the updated score.
    • Using Variables: We can use the value stored in a variable in various ways throughout our program. We can perform calculations with it, display it on the screen, or use it to make decisions. Variables allow us to work with data dynamically and flexibly.

    Here's an example code snippet in Breeze that demonstrates the use of a variable:

    set score to 10  // Declaration and assignment
out("Your score is: " + score) // Output the value of the variable

set score to score + 5 // Update the value of the variable by adding 5

out("Your updated score is: " + score) // Output the updated value

    In this example, we declare a variable called "score" and assign it an initial value of 10. We then print the value of the variable using the out function. Next, we update the value of the variable by adding 5 to it. Finally, we print the updated value of the variable. We will discuss the out function, which is used for printing messages, in more detail later.

    Variables are essential because they allow our programs to store and manipulate data dynamically. They make our programs more flexible, as we can easily change and update values as needed.

    Operators

    In Breeze, operators are symbols or special characters that allow us to perform calculations and comparisons between values. They help us manipulate data and make decisions in our programs. Breeze supports the following operators:

    Arithmetic Operators:

    • Addition (+): Adds two numbers together.
    • Subtraction (-): Subtracts one number from another.
    • Multiplication (*): Multiplies two numbers.
    • Division (/): Performs integer division, truncating any decimal portion.

    Comparison Operators:

    • Equality (=): Checks if two values are equal.
    • Greater Than (>): Checks if the value on the left is greater than the value on the right.
    • Less Than (<): Checks if the value on the left is less than the value on the right.

    These operators are used to perform various operations and comparisons within Breeze programs. They help us calculate results and compare values.

    
    set x to 5
    set y to 3
    set sum to x + y // Addition
    set difference to x - y // Subtraction
    set product to x * y // Multiplication
    set quotient to x / y // Division (integer division)

    out("Sum: " + sum)
    out("Difference: " + difference)
    out("Product: " + product)
    out("Quotient: " + quotient)

    set isGreater to x > y // Greater Than
    set isLess to x < y // Less Than
    set isEqual to x = y // Equality

    out("Is x greater than y? " + isGreater)
    out("Is x less than y? " + isLess)
    out("Are x and y equal? " + isEqual)

    In this example, we have two variables, x and y, assigned with values 5 and 3, respectively. We then use arithmetic operators to perform calculations. Then we use the out function to display the values of these variables on the screen. Next, we use comparison operators to compare the values of x and y. Finally, we use the out function again to display the results of these comparisons.

    Control Flow

    In programming, control flow refers to the order in which instructions are executed in a program. It allows us to control the flow and behavior of our code based on different conditions. Control flow statements provide mechanisms for making decisions, repeating actions, and branching between different parts of the code.

    By using control flow statements, we can make our programs more dynamic and responsive. They enable us to handle different scenarios and perform different actions based on specific conditions.

    When Statement:

    The "when" statement in Breeze allows you to control the flow of your program based on certain conditions. It provides a way to execute a block of code only if a specified condition is true. Here's the syntax of the "when" statement in Breeze:

    
    when <condition>
    do {
        // Code to be executed if the condition is true
    }
    otherwise
    do {
        // Code to be executed if the condition is false
    }

    In the "when" statement, you specify a boolean expression within the angle braces (<>) as the condition. If the condition is true, the code inside the first "do" block will run. If the condition is false, the code inside the "otherwise" block will run. This allows you to perform different actions based on whether the condition is true or false.

    Loop Statement:

    The "loop" statement in Breeze allows you to repeat a block of code a specific number of times. You can specify the number of iterations using a numeric value or a variable. Here's the syntax of the "loop" statement in Breeze:

    
    loop <number of iterations> times
    {
        // Code to be repeated
    }

    You replace <number of iterations> with the desired value. The code inside the curly braces ({}) will be executed repeatedly for the specified number of iterations. This construct is useful when you need to perform a specific action multiple times.

    While Statement:

    The "while" statement in Breeze allows you to repeatedly execute a block of code as long as a specified condition is true. Here's the syntax of the "while" statement in Breeze:

    
    while <condition>
    {
        // Code to be executed while the condition is true
    }

    The <condition> represents a boolean expression. As long as the condition remains true, the code inside the curly braces ({}) will be executed repeatedly. It's important to ensure that the condition eventually becomes false; otherwise, the loop will continue indefinitely, resulting in an infinite loop.

    These control flow statements, namely "when," "loop," and "while," provide you with the flexibility to control the flow of your program and perform actions based on conditions or repetitive tasks.

    Functions

    In programming, a function is a reusable block of code that performs a specific task or action. By encapsulating a set of instructions within a function, we can execute that code whenever needed by calling the function.

    There are two types of functions in Breeze:

    • Value Functions: Value functions in Breeze are functions that return a value as their result. They are designed to perform calculations or transformations and provide the computed value back to the calling code. Examples of value functions include functions that perform mathematical calculations, convert data types, or retrieve data from variables.
    • Action Functions: Action functions in Breeze do not return a value. Instead, they perform a specific action or task without producing a result that can be assigned or used further in the program. Action functions typically have side effects such as displaying output, modifying variables, or interacting with the user.

    Each function has the following elements:

    • Function Name: A function in Breeze has a name that uniquely identifies it. The name serves as a reference point to call the function from other parts of the program. It should be meaningful and indicative of the task or purpose the function performs.
    • Parameters: Parameters, also known as arguments, are variables or values that a function can receive as input. They allow us to provide data to the function that it needs to perform its task. Parameters are defined within the parentheses after the function name. They act as placeholders for the values that will be passed into the function when it is called.
    • Return Value (for value functions): A return value is the result or output produced by a function after executing its code. It allows a function to pass data back to the code that called it. In Breeze, some functions have a return value that can be assigned to a variable or used directly in expressions. The return value is specified using the "return" keyword within the function.

    In Breeze, there are several built-in functions available for performing various tasks. Let's go through each of the built-in functions, categorizing them into action and value functions, and explaining their purposes:

    Action Functions:

    • out: This function is used to display output or print a value to the user. It takes a single parameter and does not return a value. It is commonly used to show messages or results on the screen.

    Value Functions:

    • in: The in function is used to retrieve input from the user. It takes a single parameter of type Text that prompts the user for input. It returns the user's input as a Text value.
    • to-number: The to-number function converts a Text value representing a number into an actual numeric value of type Number. It takes a single parameter of type Text and returns the corresponding numeric value.
    • to-text: The to-text function converts a numeric value of type Number into its textual representation of type Text. It takes a single parameter of type Number and returns the equivalent text representation.

    These built-in functions in Breeze provide essential functionality to interact with the user, display output, retrieve input, and perform data conversions. You can use them to enhance the interactivity and flexibility of your programs.

    Examples

    Age Verification

    This program asks the user to enter their age and displays a message based on whether the age is greater than 18. 

    
        // Set Text messages
        set ok_msg to "Enter the Bar"
        set err_msg to "Leave now"

        // Input the age and convert to number
        set age to to-number(in("Enter age: "))

        // Check if the age is valid
        // And output the correct message
        when age > 18
        do {     
            out(ok_msg)
        }
        otherwise
        do {
            out(err_msg)
        }

    In this code, we have a program that verifies the age of a user. It prompts the user to enter their age and displays a message based on whether the entered age is greater than 18.
    The program starts by setting up two text messages: ok_msg is set to "Enter the Bar", and err_msg is set to "Leave now". These messages will be displayed depending on the age entered.
    Next, the program asks the user to enter their age using the in() function. The inputted value is then converted to a number using the to-number() function, and the result is stored in the age variable.
    After that, the program checks if the age is greater than 18 using the when statement. If the condition is true, it executes the code inside the curly braces after do, which in this case is out(ok_msg). This will display the message "Enter the Bar".
    If the age is not greater than 18, the program executes the code inside the curly braces after otherwise, which is out(err_msg). This will display the message "Leave now".
    Overall, this code demonstrates a simple age verification system that provides different messages based on the entered age.





    Celsius to Fahrenheit Conversion

    This program prompts the user to enter a temperature in Celsius, converts it to Fahrenheit, and outputs the converted temperature. 

    
        // Prompt the user to enter temperature in Celsius
        set celsius_temp to to-number(in("Enter temperature in Celsius: "))

        // Convert Celsius to Fahrenheit
        set fahrenheit_temp to celsius_temp * 9 / 5 + 32

        // Output the converted temperature in Fahrenheit
        out("Temperature in Fahrenheit: ")
        out(fahrenheit_temp)

    In this code, we have a program that converts a temperature from Celsius to Fahrenheit. It prompts the user to enter a temperature in Celsius, performs the conversion, and then outputs the converted temperature in Fahrenheit.
    The program starts by using the in() function to ask the user to enter a temperature in Celsius. The inputted value is then converted to a number using the to-number() function, and the result is stored in the variable celsius_temp.
    Next, the program performs the conversion from Celsius to Fahrenheit. It multiplies the celsius_temp by 9, divides the result by 5, and adds 32. The calculated Fahrenheit temperature is stored in the variable fahrenheit_temp.
    Finally, the program outputs the converted temperature in Fahrenheit. It uses the out() function to display the text "Temperature in Fahrenheit: ", followed by the value of the fahrenheit_temp variable.
    Overall, this code demonstrates a simple temperature conversion from Celsius to Fahrenheit. It asks the user for the temperature in Celsius, performs the conversion, and outputs the result in Fahrenheit.
